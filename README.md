## 📸 Project Overview
Captured By Kohl is an elite-tier photography portfolio and media delivery system. It is engineered to bridge the gap between professional capture and athlete accessibility. By utilizing advanced web protocols, this site provides a cinematic experience for viewing and downloading high-resolution sports media, specifically optimized for high-school basketball, track & field, and broadcast-style graphic design.

### 🛠 The Tech Stack
* **Architecture:** Static SPA (Single Page Application)
* **Engine:** HTML5 / CSS3 (Custom Masonry & Grid Logic)
* **Motion:** [GSAP](https://greensock.com/gsap/) (Advanced Smooth-Lag Cursor & Staggered Reveals)
* **Media Pipeline:** Cloudinary API for serverless asset management
* **Typography:** Syncopate & Space Grotesk via Google Fonts

---

## 🚀 Key Features

### 1. Triple-Stack Filter Logic
Navigate the archive through a multi-dimensional sorting system. Users can instantly isolate media by:
* **Domain:** Sports vs. Portraits
* **Category:** Basketball, Track & Field, or Graphic Edits
* **Archive Year:** Direct access to 2025 and 2026 seasons

### 2. The "Darkroom" HUD
* **Custom Cursor:** Adaptive circular cursor with GSAP smoothing for immersive navigation.
* **Dynamic Download Logic:** Secure fetch-blob protocol for high-resolution file retrieval.
* **Cyber-Design Aesthetic:** High-contrast monochrome UI with action-red accents and frosted-glass components.

### 3. Stealth Admin Portal
A secure, front-end management interface triggered by a hidden keyboard sequence (**Ctrl+U**), allowing for live game-day updates and cloud uploads without public clutter.

## ⚙️ Deployment Instructions

1.  **Clone the Archive:**
    ```bash
    git clone [https://github.com/capturedbykohl/capturedbykohl.github.io.git](https://github.com/capturedbykohl/capturedbykohl.github.io.git)
    ```
2.  **Media Injection:**
    Replace local image paths in `index.html` or update the Cloudinary API keys to point to your personal media bucket for live cloud hosting.
3.  **Live Push:**
    Commit changes and push to the `main` branch to trigger the GitHub Pages production deployment.
